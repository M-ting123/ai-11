核心概念

你是一个刚拿到律师证的新手律师，意外获得了AI助手系统——别人没有，只有你有。

为被冤枉的人辩护，在法庭上揭穿伪造证据与不成立的推理。


核心设定

  • 金手指：只有主角有AI助手，这是"作弊"的爽感
  • 律师身份：审查证据而非采集，情报来自警方卷宗
  • 不涉及真实法律：法庭是博弈舞台，不是法律模拟器

核心玩法流程

开始界面 ↓ 剧情黑场 + 插画 + 打字机文字 ↓ 受理案件（文件解封呈现） ↓ 连线整理（梳理卷宗物证矛盾点） ↓ 传唤证人当面询问（主玩法 · AI对话 · 搜集疑点） ↓ 疑点整理（疑点做卡片 → 连线 → 拖动摆位） ↓ 开庭（威慑 / 异议 / 出示证据） ↓ 判决 + 剧情推进

传唤是"听他说话"，开庭是"用他的话打他的脸"。不重复。

传唤询问 — 主玩法

3个操作

操作做什么效果
选话题提问选提问方向（不自由输入）证人回答，可能露出疑点
点击疑点句点击证词中可疑的句子记录到右侧"已记录疑点"
出示物证施压拿已有物证逼证人回答证人可能改口或透露更多

右侧AI助手面板

  • 上方：疑点提示（"这句话可能有问题"，不给答案）
  • 下方：已记录疑点列表（累积显示）
  • 审讯回合限制：每个证人5轮

法庭玩法 — 4个阶段

阶段内容操作
1. 证人作证揭穿证词谎言点击可疑句 → 异议 → 出示证据
2. 律师交锋反驳对方推理链找断裂点，点击反驳
3. 情绪崩溃证人被逼崩溃AI演绎情绪变化，改口承认
4. 最终质询串联证据揭真相一击揭真相，高潮收尾

胜负判定

  • 全部揭穿 = 完美胜诉
  • ≥2/3 = 胜诉
  • 1/3 = 部分胜诉
  • 0 = 败诉

AI 使用说明

1. 剧情与NPC：DeepSeek生成案件剧本、NPC人设、证词与谎言逻辑，输出JSON实现剧情代码解耦。

2. AI证人对话（核心）：DeepSeek构建Persona系统，NPC有预设人设、知识边界与谎言。AI根据角色性格生成回应，动态演绎情绪变化（冷静→紧张→崩溃），实现可揭穿的智能证人。

3. AI助手分析：右侧面板实时分析证词与证据匹配，标记疑点但不给答案，辅助玩家发现矛盾。

4. 美术：GPT-4o生成Alan Becker风格火柴人立绘与场景图片。

5. 代码：CodeBuddy生成GDScript，涵盖UI布局、拖拽连线、AI调用与状态管理。

6. API安全：EdgeOne Functions做代理，密钥不进前端。

3处矛盾（揭穿≥2处胜诉）

  • 时间矛盾：保安称21:30在岗亭 ← 监控显示21:30离开
  • 位置矛盾：保安称没去过3号楼 ← 值班记录显示21:30签到3号楼
  • 凶器矛盾：保安称看到被告手持凶器 ← 凶器无被告指纹、手套无血迹

真相：凶手是保安自己，被告被栽赃
