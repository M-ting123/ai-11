# 比赛材料

## 作品标题

《异议！》

---

## 作品描述 Project Description

**背景**：你是一个刚拿到律师证的新手律师，意外获得了AI助手系统——别人没有，只有你有。为被冤枉的人辩护，在法庭上揭穿伪造证据与不成立的推理。

**玩法**：每案分三阶段——传唤询问（选话题提问、点击疑点句、出示物证施压，搜集证人谎言）、疑点整理（将疑点做成卡片连线摆位，准备开庭弹药）、开庭对决（威慑、异议、出示证据，揭穿证词矛盾与对方律师推理漏洞）。法庭分四阶段推进：证人作证→律师交锋→情绪崩溃→最终质询，揭穿全部矛盾即完美胜诉。

**亮点**：
1. **AI证人是核心玩法**：基于DeepSeek构建Persona系统，NPC有预设人设、知识边界与谎言。AI动态演绎证人情绪变化（冷静→紧张→崩溃），每次对话回应都不同，实现"可被揭穿的智能证人"。
2. **金手指设定**：只有主角有AI助手，它标记疑点但不给答案，玩家必须自己思考判断。
3. **传唤与法庭不重复**：传唤是"听他说话"搜集弹药，开庭是"用他的话打他的脸"正式揭穿。
4. **防AI幻觉**：NPC知识和谎言预设，AI只负责台词风格不负责事实，无幻觉风险。

---

## AI 使用说明 AI Usage Description

本作在以下环节深度使用AI：

1. **剧情与NPC**：DeepSeek生成案件剧本、NPC人设、证词与谎言逻辑，输出JSON实现剧情代码解耦。

2. **AI证人对话（核心）**：DeepSeek构建Persona系统，NPC有预设人设、知识边界与谎言。AI根据角色性格生成回应，动态演绎情绪变化（冷静→紧张→崩溃），实现可揭穿的智能证人。

3. **AI助手分析**：右侧面板实时分析证词与证据匹配，标记疑点但不给答案，辅助玩家发现矛盾。

4. **美术**：GPT-4o生成Alan Becker风格火柴人立绘与场景图片。

5. **代码**：CodeBuddy生成GDScript，涵盖UI布局、拖拽连线、AI调用与状态管理。

6. **API安全**：EdgeOne Functions做代理，密钥不进前端。
