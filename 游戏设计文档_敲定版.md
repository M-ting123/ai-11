## 二、核心概念（已敲定）

### 一句话描述
你是一个刚拿到律师证的新手律师，意外获得了AI助手系统——别人没有，只有你有。为被冤枉的朋友辩护，在法庭上揭穿伪造证据与不成立的推理。

### 核心设定
- **金手指**：只有主角有AI助手，这是"作弊"的爽感
- **律师身份**：审查证据而非采集证据，情报来自警方卷宗+法医鉴定
- **辩护对象**：为被冤枉的人辩护（被告是朋友/亲友，强烈冤枉）
- **不涉及真实法律**：法庭是博弈舞台，不是法律模拟器

## 五、调查阶段（已敲定）

### 设计原则
- **不去现场跑图**（不做场景探索）
- 情报来源：**警方卷宗 + 法医鉴定报告**，以文件解封形式呈现
- 律师的工作：审查证据、找矛盾、构建辩护逻辑链
- **主玩法是传唤证人当面询问（AI对话）**，连线只是前景提要/信息梳理工具

### 调查阶段流程

```
1. 接收情报（卷宗+法医鉴定，文件解封阅读）← 信息输入
  ↓
2. 连线整理（快速梳理物证关联，发现可能的矛盾点）← 前景提要
  ↓
3. 传唤证人/当事人当面询问（AI对话）← 主玩法！搜集疑点
  ↓
4. 疑点整理（疑点做卡片→连线→拖动摆位）← 准备开庭弹药
  ↓
5. 进入开庭
```

- **下方**：已记录疑点列表（玩家在审讯中点击的可疑句子，累积显示在这里）

### 证据呈现方式
- **物证**：证据卡片（1张小图+文字描述），不做3D场景建模
- **文字证据**：通话记录、账单、邮件——纯文字卡片
- **证词**：AI NPC对话获取，记录在证词库
- **AI生图总量**：控制在6-9张（1张法院背景+1张办公室背景+2-3张NPC立绘+3-5张证据小图）

- **进入动画**：打开档案袋的动作（档案袋封面→展开→证据卡片出现）
- **背景**：棕色档案袋风格（主色 #5D4037 深棕，辅助 #8D6E63 浅棕）
- **布局**：左右均匀分栏，**左侧75%证据画布区 + 右侧25% AI助手区**（比例3:1），中间竖线分隔
- **画布层**：证据区覆盖一层透明画布（z_index=10），所有箭头连线在画布上绘制，覆盖在卡片之上
- **卡片自由拖动**：卡片可在画布上自由拖动到任意位置（不限于网格），玩家手动整理证据布局产出"位置逻辑图"
- **操作方式**：
  - 左键按住卡片拖动 = 移动卡片位置
  - 短按卡片（不拖动）= 选中为连线起点（黄色高亮边框）
  - 选中后再点击另一张卡片 = 完成箭头连线
  - 点击空白处 = 取消选中
- **连线样式**：画布风格箭头连线（粗线+大三角+起点圆点+白色描边），思维导图风格
- **证据卡片**：卡片式，可拖拽连线
- **位置持久化**：卡片位置存入 `GameState.card_positions`，审讯证人界面上方连线图复现该布局并居中显示

## 六、连线玩法（已敲定，定位为前景提要）

### 定位
连线**不是主玩法**，是前景提要/信息梳理工具。真正的信息挖掘靠传唤对话（第七节）。连线只是帮你快速看清"哪些证据有关联"，提示你去问证人这些问题。

### 核心玩法
玩家从混乱的证据卡片中，拖拽连线，把有关联的证据连起来，发现可能的矛盾点。

### 连线逻辑

```
证据卡片（乱序排列）
  ├─ 监控显示21:00离开 ──┐
  ├─ 证人说21:30离开 ────┘ → 连对=解锁「时间矛盾」
  ├─ 凶器是水果刀 ───────┐
  ├─ 嫌疑人家里有水果刀 ──┘ → 连对=解锁「凶器关联」
  └─ 嫌疑人有不在场证明 ── ← 与监控矛盾
```

### 交互
- 拖拽证据卡片，把有关联的连起来
- **连对** → 解锁"矛盾点"，可在法庭使用
- **连错** → AI助手提示"这个关联不太对"
- **右侧AI助手**可给提示（无消耗）

### 技术实现
- Godot的Control节点原生支持拖拽
- 连线用Line2D或draw绘制
- 判定：预设正确关联对，玩家连线匹配即触发

## 七、传唤证人（已敲定）

### 设计
- 传唤当事人/目击证人，进行AI对话
- AI NPC有完整Persona（人设+知识边界+谎言设定）
- 玩家选择提问方向，NPC根据人设回答
- AI助手标记证词中的可疑点

### AI NPC约束（防幻觉）

```json
{
  "npc_id": "witness_01",
  "persona": "紧张、回避眼神、说话结巴",
  "knowledge_boundary": {
    "knows": ["案发时间21:00", "被害人在办公室"],
    "does_not_know": ["凶手身份"],
    "lies_about": ["离开时间（谎称21:30，实际21:00）"]
  },
  "response_rules": {
    "被问及离开时间": "坚持说21:30，表现紧张",
    "被出示监控证据": "崩溃，承认21:00离开",
    "被问及凶手": "表示不知道（真不知道）"
  }
}
```

- NPC的"知识"和"谎言"是**预设的**
- AI只负责"怎么表达"（台词风格），不是"事实内容"
- 超出知识边界的问题，NPC统一回答"不知道/不记得"
- 这样不会有幻觉——AI没有编造事实的权限

### 传唤询问的3个操作（v0.7更新）

| 操作 | 做什么 | 效果 |
|:---|:---|:---|
| **提问** | 选提问方向→选具体问题→AI回答 | 获取证人证词 |
| **整理** | 当前证词变卡片→连线证据 | **连接成功=正常（证词与证据一致）；连接失败=疑点（证词与证据矛盾）**；连接错误提示再试试 |
| **进入法庭** | 审讯结束 | 携带已找到的疑点进入法庭 |

**v0.7 审讯内整理机制（核心创新）**：
- 每询问一个问题后，玩家可点[整理]按钮
- 当前证词变成**疑点卡片**（颜色区分于证据卡片，如橙红色边框）
- 玩家把疑点卡片与证据卡片连线
- 系统判定：
  - **证词与证据一致**（印证关系）→ 连接成功，绿色，正常
  - **证词与证据矛盾**（矛盾关系）→ 连接失败，标记为**疑点**，右侧"疑点已找到 ✓"
  - **连接错误**（无关联）→ 提示"再试试"
- 疑点卡片颜色区分度：橙红色边框（证据卡片是棕色边框）

**AI 右侧界面软提示**：
- 当AI回答有问题时，右侧软提示"好像有些矛盾？"
- 玩家可点击提示获得指引（建议去整理连线）
| **选话题提问** | 选提问方向（不是自由输入） | 证人回答，可能露出疑点 |
| **点击疑点句** | 证人说完后，点击你认为有问题的句子 | 记录到右侧"已记录疑点"列表 |
| **出示物证施压** | 拿已有物证逼证人回答 | 证人可能改口或透露更多信息 |

**审讯回合限制**：每个证人有**5轮** `[PLACEHOLDER]` 审讯机会，用完证人拒绝继续配合。不能无限问，制造紧张感但不punishing（物证在法庭也能翻盘）。

**为什么不让玩家自由输入提问**：
- 避免玩家在对话框里把案子聊穿了，跳过法庭
- AI回答不可控，可能给出不该给的信息
- 选话题+点句子既有限制，又有操作感

## 七-2、法庭前疑点整理（v0.5新增，已敲定）

### 定位
审讯后、开庭前的**准备环节**。把审讯搜集的疑点做成卡片，连线+拖动摆位，为法庭做弹药准备。

### 和审讯前连线的区别

| 环节 | 目的 | 操作对象 |
|:---|:---|:---|
| **审讯前连线** | "我有哪些信息" | 卷宗物证 + 已知证词 |
| **审讯后整理** | "我要怎么用这些信息打人" | 疑点卡片 + 物证卡片 |

两个环节目的不同，不重复。

### 具体操作
1. 审讯时记录在右侧的"已记录疑点"→ 自动变成卡片拖到整理界面
2. 疑点卡片可以和物证卡片连线（标记"这个疑点和这个物证有关"）
3. 卡片可以拖动摆位（方便法庭时快速找到）
4. 这一步是"排兵布阵"，让玩家在进入法庭前理清思路

### 技术实现
- 复用连线玩法的拖拽+连线机制（第六节）
- 疑点卡片数据从审讯记录读取
- 整理结果存入 `GameState.battle_prep`，法庭界面可调用

## 八、法庭玩法（v0.7更新 · 四阶段重写）

### 法庭四阶段总览

| 阶段 | 内容 | 玩法 |
|:---|:---|:---|
| **阶段1：证人作证** | 揭穿证词谎言 | 异议 + 出示疑点/证据 |
| **阶段2：律师交锋** | 反驳对方推理链 | 找断裂点，点击反驳 |
| **阶段3：情绪崩溃** | 证人被逼到崩溃 | AI演绎情绪变化，改口承认 |
| **阶段4：最终质询** | 串联证据揭真相 | 证据按顺序排列，一击揭真相 |

> **教学关（关卡1）弱化律师**：律师不与玩家交锋，只是将证人错误证词串联出"所谓真相"的人。情绪崩溃与最终质询环节照常进行。

### 教学关法庭流程（关卡1）

1. **证人陈述**：证人以自己的说法讲述来龙去脉（声称看见了什么、自己根本没在场等），实际是在撒谎栽赃被告
2. **律师还原**：对方律师假惺惺地把证人证词串联成"所谓真相"，主张被告有罪
3. **逐句质询**：系统重放证词，每次给出一句话，玩家可选择：
   - **异议**：需使用审讯阶段搜集的「疑点」提出（疑点已绑定证据与矛盾）
   - **下一句**：跳过当前句
4. **情绪分级反应**（按已点出矛盾数量）：
   - 点出 **1处**：证人狡辩，情绪不太对，有点不正常
   - 点出 **2处**：证人说不出话，甚至说出逻辑混乱的话
   - 点出 **3处**（全部）：证人失声痛哭，绝望求饶
5. **胜负门槛**：**必须3处全点出**才能进入真相揭示并胜诉；只点出2处无法拼凑出完全的真相
6. **法官发问**：3处全揭穿后，法官敲响法槌询问"那么真相是什么？"
7. **真相揭示**：玩家"生成"真相陈述（打字机逐字缓慢呈现），得出真实答案——真凶就是证人本人
8. **胜利宣告**：火柴人（玩家）比出逆转裁判经典指认手势，宣告胜利

```
│       法庭主交互区（左侧）      │  AI助手面板（右侧）│
│                                │                  │
│  - 阶段标题（当前所处阶段）     │  - 疑点提示       │
│  - 说话人 + 证词/陈述文本       │  - 建议质询方向   │
│  - 已揭穿矛盾 X/3 · 异议剩余   │  - 局势评估       │
│                                │                  │
├────────────────────────────────┴──────────────────┤
│              操作栏（底部）                        │
│            [异议]      [下一句/继续]              │
└───────────────────────────────────────────────────┘
```

### 操作（2种）

| 操作 | 触发条件 | 效果 | 失败后果 |
|:---|:---|:---|:---|
| **异议** | 证词重放阶段，当前句可异议 | 弹出疑点列表，选择匹配疑点揭穿矛盾，触发证人情绪反应 | 疑点不匹配→浪费1次异议机会 |
| **下一句/继续** | 任何时候 | 推进到下一句证词 / 进入下一阶段 | 跳过可异议句则该矛盾无法再揭穿 |

> 旧版「威慑」「出示证据」操作在四阶段重做后合并入「异议」（用疑点=证词+证据组合）。

### AI助手的提示原则（已敲定）
- AI助手**绝对不能直接说正确答案**
- AI助手只能说"这里有疑点"、"这句话可能有问题"、"我感觉到不对劲"
- 玩家必须自己思考，决定点哪句话、用哪个疑点
- AI助手是"提醒你有问题"，不是"告诉你答案"
- 这样保持玩家的参与感和成就感

### 胜负机制：关键矛盾制（v0.7四阶段）

**核心原则**：胜负只看是否在质询阶段点出全部矛盾。

- 法庭质询阶段逐句重放证词，可异议句对应一处矛盾
- 玩家用审讯阶段搜集的疑点提出异议，匹配则「点出」该矛盾
- 证人情绪随点出数量分级反应（狡辩→混乱→崩溃）

```
关键矛盾点出 = 3/3（全部）→ 证人崩溃 → 真相揭示 → 完美胜诉
关键矛盾点出 < 3 → 无法拼凑完全真相 → 部分胜诉/败诉
关键矛盾点出 = 0 → 败诉
```

**关卡难度梯度**：
- 关卡1（教学）：3个矛盾必须全部点出才胜诉（教学关引导找齐3个疑点）
- 关卡2（主线）：3个矛盾必须全部揭穿，且律师会反击（阶段2交锋）

### 证人情绪反应（阶段3，AI演绎）
- 证人情绪随被点出的矛盾数量实时变化
- 教学关用预设情绪文本（狡辩/混乱/崩溃三档）
- 主线案可接入AI实时演绎证人情绪变化与改口

### 法官（预设脚本，非AI）
- 只在关键节点说话（开场、最终发问"真相是什么"、判决）
- 3处矛盾全揭穿后敲法槌，询问真相
- 判决基于规则：矛盾是否全部点出 → 胜诉/败诉

### 对方律师（预设行为，非AI）
- 教学关：不交锋，只陈述"所谓真相"串联错误证词
- 主线案：会反击，推理链有逻辑漏洞，需玩家异议反驳（阶段2交锋）

## 九、AI结合方案（已敲定）

### 两个AI角色

| AI角色 | 位置 | 功能 | 技术 |
|:---|:---|:---|:---|
| **AI助手** | 右侧面板 | 矛盾检测、建议方向、局势评估 | DeepSeek API |
| **AI证人** | 对话界面 | 接受询问，有谎言，可被揭穿 | DeepSeek API + Persona约束 |

### AI助手功能（无算力限制，已敲定删除算力点系统）

| 功能 | 说明 |
|:---|:---|
| **疑点提示**（被动） | 常驻，自动标记证词中"可能有问题"的地方，但不给答案 |
| **建议方向** | 推荐追问方向，只给方向不给具体问题 |
| **局势评估** | 显示当前已揭穿矛盾数 / 目标数 |

- **无算力消耗**，AI助手功能可随时使用
- 但AI只给提示和疑点，不给答案，玩家仍需自己思考

### AI调用架构（2个模块）

```
模块1：AI证人对话生成
  ├─ 输入：角色人设 + 知识边界 + 谎言设定 + 玩家提问
  ├─ 模型：DeepSeek
  └─ 输出：符合角色的回应（不超出知识边界）

模块2：AI助手分析
  ├─ 输入：当前证词库 + 证据库
  ├─ 模型：DeepSeek
  └─ 输出：矛盾列表 + 建议方向 + 局势评估
```

### API密钥安全
- Godot导出Web后代码可见，密钥不能放前端
- 用EdgeOne Functions做代理中转
- Godot调用代理地址 → 代理调DeepSeek → 返回结果
